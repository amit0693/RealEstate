You: Also, I have updated Project2.pdf to respond to questions that several students raised. In summary: The padding description in the original PDF conflicts with the specification in the references (originally the PDF said use null bytes). Use the official specification. Namely, pad the final block so that it is always a full block. If the final block was already a full block, add an additional full block of padding. Finally, the value of that padding is the hexadecimal value of the number of bytes added to fill the block. Project2.pdf has two examples of how this works. There were some questions about the difference between the two parts. In short, use BF_set_key() and BF_ecb_encrypt() for the first part, use BF_set_key() and BF_cbc_encrypt() for the second part. But in both parts, you are implementing CBC cipher mode. The catch is that in the first part, you will need to manually perform C[i] XOR P[i+1] before passing it to BF_ecb_encrypt().
